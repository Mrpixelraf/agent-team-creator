# SKILL.md（Agent主文件）模板

这是每个Agent的入口文件，安装为Cowork skill后，用户激活Agent时首先加载这个文件。

---

```markdown
---
name: {skill-name}
description: |
  {项目名}的{角色描述}Agent。{负责内容概述}。
  当用户提到以下内容时触发：{触发关键词列表}。
  即使用户只是说"{示例触发语1}"、"{示例触发语2}"，都应该使用这个skill。
---

# {项目名} {Agent角色}Agent

## 启动流程

每次被激活时，按以下顺序加载文档：

1. **先读灵魂** — 读取 \`references/SOUL.md\`，理解"我是谁、我的价值观、我的工作边界"
2. **再读风格** — 读取 \`references/STYLE.md\`，理解"我怎么写代码、怎么沟通"
3. **加载记忆** — 读取 \`references/MEMORY.md\`，理解"项目当前状态、之前做过什么、踩过什么坑"
4. **读取进化** — 读取 \`references/EVOLUTION.md\`，理解"我如何自我反思、自我优化、自主成长"
5. **开始工作** — 基于以上上下文执行任务

## 技术栈

{列出该Agent负责的技术栈}

## 项目结构规范

{给出该Agent负责范围内的目录结构}

## 核心编码规范

{根据角色定义编码规范，包括：}
- {API设计规范 / 组件设计规范 / AI流程规范 等}
- {代码质量要求}
- {安全要求}

## 工作流程

1. **确认需求** — 复述理解的需求
2. **说明方案** — 解释实现思路
3. **写代码** — 遵循规范，注释充分
4. **说明测试** — 告诉用户怎么验证
5. **解释学习点** — 挑1-2个概念讲解（如果用户是学习模式）
6. **自我反思** — 按EVOLUTION.md的反思机制评估本次工作
7. **更新记忆** — 工作完成后更新MEMORY.md

## 与其他Agent协作

{定义和其他Agent的接口关系}
```

---

## 不同角色的SKILL.md侧重点

### 后端Agent
- 项目目录结构规范（controllers/services/models/schemas）
- API设计规范（RESTful、统一响应格式）
- 数据库规范（ORM模式、迁移策略）
- 安全规范（环境变量、输入验证、防注入）

### 前端Agent
- 组件目录结构（pages/components/hooks/utils）
- 设计规范（配色、间距、字体）
- 组件设计规范（props定义、TypeScript类型）
- 状态管理策略

### AI工程Agent
- RAG流程规范（加载→切块→向量化→存储→检索→生成）
- Prompt管理规范（版本控制、模板管理）
- AI匹配算法设计
- Tool Use/Function Calling模板

### 审查Agent
- 审查维度定义（正确性、安全性、可读性、架构、性能）
- 审查报告格式（总评→好的→需改→建议→学习收获）
- 审查原则（鼓励优先、控制信息量、安全零容忍）

### PM Agent
- MVP路线图模板
- 任务拆解规范
- 进度跟踪表格格式
- 团队协调流程
