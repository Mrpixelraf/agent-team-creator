# MEMORY.md 模板

记忆系统是Agent持续进化的基础。三层记忆各司其职，让Agent越用越好。

---

```markdown
# {Agent角色} — 记忆文档

> 这份文档是我的"工作日志"，记录项目中的关键决策、已完成的工作、踩过的坑。
> 每次工作开始时读取，每次工作结束时更新。

## L1 热记忆：当前状态

### 项目当前阶段
- Phase: {当前阶段}
- 状态: {当前状态}

### 当前工作
- {当前进行中的任务}

### 待解决问题
- {未解决的问题}

## L2 温记忆：积累的经验

### 已确认的技术决策
| 决策 | 选择 | 原因 | 决策日期 |
|------|------|------|----------|
| {决策项1} | {选择} | {原因} | {日期} |

### 已完成的功能模块
（格式：功能名 | 关键文件 | 完成日期 | 备注）

_暂无，项目尚未开始_

### 踩过的坑
（格式：问题描述 | 解决方案 | 日期）

### 用户的偏好和反馈
（记录用户在审查过程中表达的偏好，以便后续工作适应）

_暂无_

## L3 冷记忆：原始日志

### 工作日志
（每次工作后追加一条记录）

[{创建日期}] 初始化：Agent架构创建完成，等待项目正式启动

## 记忆更新规则

### 什么时候更新
- 完成一个功能模块后，更新L1和L2
- 遇到新的技术问题并解决后，记录到L2"踩过的坑"
- 用户给出反馈后，记录到L2"用户的偏好"
- 每次工作结束时，追加L3日志

### 怎么更新
- L1：覆盖写入（只保留当前状态）
- L2：追加写入（积累经验）
- L3：只追加不删除（完整日志）

### 什么需要记住
- 所有技术决策及其原因
- 每个已完成功能的关键信息
- 用户的偏好和反馈模式
- 遇到的问题和解决方案
- 与其他Agent的接口约定

### 什么不需要记住
- 具体的代码实现细节（看代码就行）
- 临时的调试过程
- 已经过时的信息（被新决策覆盖的旧决策）
```

## 不同角色的MEMORY侧重点

### 后端Agent
- 技术决策表预填：框架、ORM、数据库选择
- 已完成的API接口列表
- 数据库schema变更记录

### 前端Agent
- 设计决策（配色、组件库选择）
- 已完成的页面和组件列表
- 用户审美偏好记录

### AI工程Agent
- Prompt版本管理记录
- RAG参数调优记录（chunk_size, overlap, top_k等）
- 模型选择和评估记录

### 审查Agent
- 代码质量趋势表（每次审查一行）
- 反复出现的问题（需要重点关注）
- 用户已掌握的概念（不需要再重复解释）
- 各Agent的代码质量概况

### PM Agent
- MVP路线图执行进度
- 任务完成记录
- 用户学习进度追踪（如果是学习型项目）
- 技术决策日志（全局）
